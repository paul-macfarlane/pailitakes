# Epic: Admin & Authoring (ADM)

Content lifecycle: editor, drafts, preview, thumbnails, scheduling, and the cron revalidation endpoint. Ref: FR-7.x, FR-1.5/1.6; technical-design.md §4 (visibility-as-query), §5.7.

- [x] **ADM-1** — Admin route group `/admin/**`: middleware role gate + `noindex`. Implemented as cookie-presence proxy + `requireStaff()` in layout and every admin page (ADR-0009). _(deps: FND-4)_
- [x] **ADM-2** — Post editor: Markdown input + toggleable preview using the _same_ pipeline as production (server action returns rendered HTML); autosave drafts on interval. Thumbnail/banner/video URL fields deferred to ADM-6. _(deps: POST-3, ADM-1)_
- [x] **ADM-3** — Post CRUD server actions with session+role+ownership checks; `revalidateTag` on mutate. Hard delete is admin-only (authors archive, ADM-4); drafts may hold `""` thumbnail until publish-time validation (ADM-4/6). _(deps: POST-2, ADM-1)_
- [x] **ADM-4** — Draft status + status transitions (draft/scheduled/published/archived); archive is recoverable. `transitionPostStatus` action + pure `post-status.ts` state machine + `PostStatusControls` on the edit page. Publish-time thumbnail gate enforced here → editor-created posts can't be published until a thumbnail is set (ADM-6 adds the field). Moving INTO `scheduled` (future timestamps) is ADM-5. _(deps: ADM-3)_
- [x] **ADM-5** — Schedule publish/archive: `publish_at`/`archive_at` fields; visibility becomes automatic via the query predicate. `schedulePublish`/`scheduleArchive`/`cancelScheduledArchive` actions + `PostScheduleControls` (datetime-local island); cross-field CAS guards; publish-time thumbnail invariant now also enforced in `updatePost`. Exact cache invalidation at crossing time is ADM-9's cron. _(deps: ADM-4)_
- [x] **ADM-6** — Thumbnail URL field; validate `https://` image; render `next/image` `unoptimized` with explicit dimensions. Also the optional **banner URL** field (`posts.banner_url`, POST-9) with the same https validation. Editor fields + live `ExternalImage` previews; client validation reuses server `httpsImageUrl`; published/scheduled posts require a thumbnail client-side (mirrors ADM-5 invariant, prevents an autosave wedge). _(deps: ADM-2)_
- [x] **ADM-7** — Preview route `/admin/preview/[id]` — renders any draft/scheduled post in the public layout, auth-gated. Shared `PostArticle` (extracted from the public post page → pixel-identical) + `getPostForPreview` (bypasses visibility, ownership-scoped); banner uses `isPubliclyVisible` (real predicate, not raw status); Preview link on the editor. _(deps: ADM-2, POST-5)_
- [x] **ADM-8** — Dashboard post list with filter (status/category/author) + sort; authors see own, admin sees all. Server-rendered with URL-param filters (ADR-0010, not TanStack Query); `listAdminPosts` hard-scopes non-admins to their own rows; filters sanitized against option lists; Prev/Next pagination. _(deps: ADM-3)_
- [x] **ADM-9** — Cron endpoint `/api/cron/revalidate` (cron-job.org target): `CRON_SECRET` bearer auth (timing-safe), idempotent, DB-tracked last-run (`revalidation_state` singleton, migration 0003/0004). Revalidate-before-advance (at-least-once) + monotonic marker; scoped to scheduled/published crossings. _(deps: ADM-5)_
- [x] **ADM-10** — Admin user management: list users, assign roles (reader/author/admin), ban/unban (FR-4.8, FR-10.2; technical-design.md §5.7 users screen). Admin-only (`requireAdmin`); single "never leave zero active admins" invariant enforced under an admin-set lock that serializes concurrent removals; shared role values/labels; user indexes (migration 0005). _(deps: ADM-1)_
