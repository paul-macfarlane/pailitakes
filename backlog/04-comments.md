# Epic: Comments (CMT)

Nested comment threads with LLM moderation and rate limiting. Ref: FR-4.x; technical-design.md §5.2, §5.3.

- [x] **CMT-1** — `comments` schema: `parent_id` self-FK, `status` (visible/held/rejected/deleted), `mod_verdict` jsonb, indexes `(post_id, created_at)` + `(status, created_at)`. Also `(author_id, created_at)` (rate-limit counts) and `(parent_id)` (delete child-check); migration 0010. _(deps: POST-1)_
- [x] **CMT-2** — Comment tree read route `GET /api/comments?postId=` (`no-store`); one query, tree assembled in memory by `parent_id`. Fetches all statuses with server-side redaction (generalized placeholder rule, ADR-0020); response `meta` carries `comments_locked` so lock toggles never revalidate the cached post page. _(deps: CMT-1)_
- [x] **CMT-3** — Comment tree UI (TanStack Query): nested render, indent to ~depth 5 then flatten with "replying to @name", mobile-first; optimistic insert on allow. Banned viewer sees an explicit notice in place of the composer ("You're banned from commenting"), not a failing input — they can still read the thread (FR-4.8; §350). _(deps: CMT-2, CMT-4; scope added 2026-07-09 by Paul)_
- [x] **CMT-4** — Create-comment server action: reject if banned/archived/locked, then rate limit, moderate, insert with resulting status. Rejection returns an explicit reason (banned/archived/locked) the UI can surface — not a generic error swallowed at the boundary. Typed `CommentDenialReason` + `CommentSubmitResult`; reply-vs-hard-delete race classified, not thrown. _(deps: CMT-5, CMT-6; scope clarified 2026-07-09 by Paul)_
- [x] **CMT-5** — Rate limiting lib: Postgres counts by `author_id` (>3/min, >30/hour; values from config). Edits count toward the same budget + per-comment 60s edit cooldown (ADR-0020). _(deps: CMT-1)_
- [x] **CMT-6** — Moderation lib `src/lib/moderation.ts`: AI SDK → Vercel AI Gateway → `anthropic/claude-haiku-4.5`; strict JSON verdict, family-friendly policy prompt, few-shot example set in repo, ~5s timeout, fail-closed to `held`. Lives at `src/lib/comments/moderation.ts` per the ADR-0013 domain layout (recorded in ADR-0020); comment text delimiter-escaped against prompt injection. _(deps: FND-5)_
- [x] **CMT-7** — Edit/delete own comment; deleting a comment with replies leaves a `[deleted]` placeholder. Edits are re-moderated; childless deletes are hard, soft deletes clear the body (ADR-0020). _(deps: CMT-3)_
- [x] **CMT-8** — Admin: delete any comment; lock comments on a post — adds `comments_locked` to `posts` (FR-4.4, ADR-0004). Column pre-existed (ADR-0004); added `Action.ManageAnyComment`, the delete-any control in the thread, and an admin-only lock toggle on the edit page. _(deps: CMT-3, ADM-1)_
- [x] **CMT-9** — Moderation log screen: browse held/rejected with verdict + reason; approve `held`, restore `rejected` false positives. `/admin/moderation`, `Action.ModerateComments`, CAS status transitions. _(deps: CMT-4, ADM-1)_
- [x] **CMT-10** — Auto-ban repeat moderation offenders: when a user's count of currently-`rejected` comments within a rolling window crosses a threshold, set `banned_at` automatically. Suggested default: 5 rejected in 7 days, both env-configurable alongside the rate-limit values; `held` never counts (LLM failures aren't violations); an admin restore of a false positive decrements the count (count `rejected` rows live, don't keep a separate tally); auto-bans surface in the moderation log so admin can review/unban. Needs a product-doc FR + technical-design §5.2 touch when picked up. _(deps: CMT-9; added 2026-07-12 from Paul's feedback)_
